需求：跨平台终端待办事项管理器，零外部数据库，纯本地 JSON 文件存储。

背景：组里同学日常都在终端里工作，oh-my-zsh 自带的 todo plugin 实在太简陋——只有加、勾、列三个动作。我们想搞一个稍微"业务化"一点的：除了增删改查，还要能按优先级排序、按分类标签过滤、每周自动出一份完成情况周报。最烦的就是开会时心急连续敲同一行命令，结果同一个待办被加了五六次，希望能有个"短时间重复输入去重"的小规则。

核心功能：
- 任务增删改查：每条任务带标题、优先级（P0/P1/P2/P3）、分类标签（工作/学习/生活/自定义）、截止时间、可选备注
- 5 分钟内重复添加同标题任务自动去重为一条，但超过 5 分钟后再加同标题视为新任务（不打断用户后续新增同名工作的需求）
- 列表视图支持按优先级、截止时间、创建时间三种排序，分类标签多选过滤
- 周报视图：自动汇总本周已完成 vs 未完成，按分类统计耗时占比，输出到终端
- 任务归档：已完成超过 30 天的自动移到归档区，列表默认不展示；归档区可手动查看
- 数据持久化：JSON 文件存 `~/.gocli-todo/tasks.json`，启动加载，修改立即写回；配置文件 `~/.gocli-todo/config.yaml` 支持自定义存储路径、归档天数、颜色主题
- 命令补全：bash/zsh/fish 各自生成对应 completion 文件，install 子命令自动写入 `~/.zshrc` 或 `~/.bashrc`

涉及模块：
- `main.go`：CLI 入口（cobra），分发 add/list/done/undone/edit/week/archive/config/completion 子命令
- `task.go`：任务结构体、CRUD、去重判断、软删除标记
- `storage.go`：JSON 读写、文件锁（flock）防止多进程并发写、原子重命名
- `report.go`：周报统计逻辑、归档判定
- `config.go`：YAML 配置文件加载、默认值、用户主题配置

注意事项：
- 任务 ID 用单调递增整数即可，重启后从 JSON 里读 max+1 继续递增；UUID 反而让命令行复制粘贴不友好
- 优先级排序时 P0 > P1 > P2 > P3，同优先级再按截止时间升序（无截止时间排最后）
- 删任务用"软删除"（`deleted: true` + `deleted_at` 时间戳），30 天后才物理删除，避免误删
- 文件锁要兼容 Windows（golang.org/x/sys/windows 的 LockFileEx）和 Unix（syscall.Flock），别直接用 syscall.Flock 否则 Windows 直接挂
- 写 JSON 时先写临时文件再 rename，避免写到一半崩溃导致主文件损坏
- 配置文件不存在时不要报错，用默认配置继续跑；写回时保留用户已设置的字段，不要全量覆盖
